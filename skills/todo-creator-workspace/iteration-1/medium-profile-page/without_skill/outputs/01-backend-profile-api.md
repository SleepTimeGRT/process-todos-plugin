# Add Backend API Routes for User Profile

This ticket implements the server-side foundation for the user profile feature: extending the Prisma schema to store profile data, and creating Next.js API routes that the frontend will call for avatar upload, nickname update, and password change. All API routes live under `src/app/api/profile/`.

- [ ] Add `avatarUrl` (String, optional) and `nickname` (String, optional) fields to the `User` model in `prisma/schema.prisma`, then run `npx prisma migrate dev --name add-profile-fields` to apply the migration
- [ ] Create `src/app/api/profile/route.ts` with a `GET` handler that returns the authenticated user's `id`, `email`, `nickname`, and `avatarUrl` from the database using Prisma, and a `PATCH` handler that accepts `{ nickname: string }` in the request body, validates that nickname is non-empty and at most 30 characters, updates the record in Prisma, and returns the updated user object
- [ ] Create `src/app/api/profile/avatar/route.ts` with a `POST` handler that accepts a `multipart/form-data` request containing a file field named `avatar`, validates that the file is an image (MIME type `image/jpeg`, `image/png`, or `image/webp`) and is under 5 MB, saves it to `public/uploads/avatars/<userId>.<ext>`, updates `User.avatarUrl` in Prisma to `/uploads/avatars/<userId>.<ext>`, and returns `{ avatarUrl }` in the response
- [ ] Create `src/app/api/profile/password/route.ts` with a `POST` handler that accepts `{ currentPassword: string, newPassword: string }`, verifies `currentPassword` against the stored hash using bcrypt, validates that `newPassword` is at least 8 characters, hashes the new password with bcrypt (salt rounds: 10), updates `User.password` in Prisma, and returns `{ success: true }` on success or a 400/401 error with a descriptive message on failure
- [ ] Add authentication guard middleware to all three route files: check for a valid session (using whatever auth library the project uses — inspect existing API routes to match the pattern), and return a 401 response with `{ error: "Unauthorized" }` if no valid session is found
- [ ] Write a `src/lib/profile.ts` utility module that exports typed helper functions used by the routes: `getUserProfile(userId: string)`, `updateNickname(userId: string, nickname: string)`, `updateAvatarUrl(userId: string, avatarUrl: string)`, and `updatePassword(userId: string, hashedPassword: string)`, each wrapping the corresponding Prisma calls
