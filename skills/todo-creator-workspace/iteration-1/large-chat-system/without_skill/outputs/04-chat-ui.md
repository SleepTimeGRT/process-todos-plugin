# Build chat UI components and pages

This ticket builds the user-facing chat interface: a sidebar listing conversations, a main message thread view, a message input field, and the routing/layout that ties them together. The UI connects to the REST API for initial data load and to the WebSocket server for real-time updates. Assumes the DB schema (01), WebSocket server (02), and API routes (03) are all in place.

- [ ] Create `src/app/chat/layout.tsx` — a two-column layout with a fixed-width left sidebar (`ConversationList`) and a flex-grow right panel (the `{children}` slot for the active conversation). Apply a `min-h-screen` wrapper so the layout fills the viewport.
- [ ] Create `src/app/chat/page.tsx` — the chat index page shown when no conversation is selected. Display a prompt like "Select a conversation to start chatting" centered in the right panel.
- [ ] Create `src/app/chat/[conversationId]/page.tsx` — the active conversation page. Server-fetch the conversation metadata (using the `GET /api/conversations/[conversationId]` endpoint via `fetch` in a server component) and pass it as props to the `<ChatWindow>` client component.
- [ ] Create `src/components/chat/ConversationList.tsx` (client component). Fetch the conversation list on mount via `GET /api/conversations`. Render each item showing: conversation name (or the other user's name for DIRECT), last message preview truncated to 60 chars, relative timestamp (e.g. "2m ago"), and an unread badge count (derive from messages after `lastReadAt`). Clicking an item navigates to `/chat/[conversationId]` using Next.js `router.push`. Highlight the active conversation.
- [ ] Create `src/components/chat/NewConversationModal.tsx` (client component). A modal triggered by a "New Chat" button in the sidebar. Contains a user search input (debounced `GET /api/users?query=...` — implement this endpoint too in `src/app/api/users/route.ts` to search users by name/email, excluding the current user). Below the search, show a toggle for `DIRECT` vs `GROUP` mode. For GROUP mode, allow selecting multiple users and entering a group name. On submit, call `POST /api/conversations` and navigate to the new conversation.
- [ ] Create `src/app/api/users/route.ts`. Implement `GET` with a `?query=` param to search users by `name` or `email` (case-insensitive contains). Return `[{ id, name, image }]`. Limit to 20 results. Require authentication.
- [ ] Create `src/components/chat/ChatWindow.tsx` (client component). On mount, fetch the last 50 messages via `GET /api/conversations/[conversationId]/messages`. Open a WebSocket connection to `ws://localhost:3000` (or `wss://` in production, read from `NEXT_PUBLIC_WS_URL` env var). On `chat:message` events, append the new message to local state. Scroll to bottom on new messages, but only if the user is already near the bottom (within 100px) to avoid hijacking scroll when reading history. On unmount, close the WebSocket connection.
- [ ] Create `src/components/chat/MessageList.tsx`. Renders the list of messages. Groups consecutive messages by the same sender into visual clusters (no repeated avatar). Each message bubble shows: sender avatar + name (for first in cluster), message content, and timestamp on hover. For messages from the current user, align right and use a distinct background color.
- [ ] Create `src/components/chat/MessageInput.tsx` (client component). A `<textarea>` that grows up to 5 lines. On `Enter` (without Shift), send the message by emitting `{ type: 'chat:send', conversationId, content }` over the WebSocket. Show a character counter when approaching 2000 chars. Disable the send button and show a spinner while the message is in flight.
- [ ] Create `src/components/chat/MemberList.tsx` — a collapsible panel (toggle with a button in the header) showing all conversation members with their avatars and names. Shown only for GROUP conversations.
- [ ] Add a `NEXT_PUBLIC_WS_URL` entry to `.env.local.example` (and `.env.local` if it exists) with value `ws://localhost:3000`.
