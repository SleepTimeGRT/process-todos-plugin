# Implement read receipt indicators in the chat UI

Read receipts make the chat feel alive — senders can see who has read their messages, and conversations show unread counts to surface new activity. The backend persistence and WebSocket events are already in place from earlier tickets; this ticket wires them into the UI and completes the end-to-end read receipt flow.

- [ ] In `src/components/chat/ChatWindow.tsx`, add an `IntersectionObserver` that watches each rendered `<MessageBubble>` for visibility. When a message from another user scrolls into view, emit `{ type: 'chat:read', conversationId, messageId }` over the WebSocket. Debounce emissions so rapid scrolling doesn't flood the server — coalesce reads within a 500ms window and send the most recent `messageId` (since reading a later message implies reading all prior ones).
- [ ] In `ChatWindow.tsx`, listen for incoming `{ type: 'chat:read', conversationId, messageId, userId }` WebSocket events and update local message state to mark the message (and all messages before it in the conversation) as read by that user.
- [ ] Create `src/components/chat/ReadReceipts.tsx`. Receives `readBy: { userId: string; name: string; image: string | null; readAt: string }[]`. Renders small stacked avatar thumbnails (max 3, with a "+N" overflow label) directly below the message bubble, aligned to the trailing edge. Add a tooltip on hover listing each reader's name and relative read time.
- [ ] Render `<ReadReceipts>` inside `src/components/chat/MessageList.tsx` for each message where `readBy.length > 0` AND the message was sent by the current user (other users' read receipts are shown only on your own messages).
- [ ] In `src/components/chat/ConversationList.tsx`, compute the unread count for each conversation as the number of messages with `createdAt > lastReadAt`. Display as a filled badge on the conversation list item. Update the badge reactively when a `chat:read` event arrives for the current user in that conversation (set `lastReadAt` in local state to the received `messageId`'s `createdAt`).
- [ ] On conversation selection (`/chat/[conversationId]` navigation), immediately emit a `chat:read` event for the latest message in the conversation so the unread count clears as soon as the user opens the chat. Do this inside `ChatWindow.tsx`'s mount effect, after the initial messages are fetched.
- [ ] Add an end-to-end manual test checklist comment at the top of `src/components/chat/ReadReceipts.tsx` describing how to verify the feature works: open two browser windows logged in as different users, send a message, observe the read receipt appear when the second user scrolls the message into view.
