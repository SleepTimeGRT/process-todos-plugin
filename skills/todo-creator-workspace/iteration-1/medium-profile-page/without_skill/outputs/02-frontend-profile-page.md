# Build the User Profile Page UI

This ticket builds the frontend for the user profile feature on top of the API routes created in `01-backend-profile-api.md`. The profile page lives at `/profile` in the Next.js App Router and is composed of three independent sections — avatar, nickname, and password — each backed by its own form component. All components are written in TypeScript.

- [ ] Create `src/app/profile/page.tsx` as a server component that fetches the current user's profile from `GET /api/profile` on the server side, then renders a page layout with the heading "내 프로필" and includes the three client components: `<AvatarUpload>`, `<NicknameForm>`, and `<PasswordForm>`, passing the fetched `avatarUrl` and `nickname` as props
- [ ] Create `src/components/profile/AvatarUpload.tsx` as a client component (`"use client"`) that displays the current avatar using `<Image>` from `next/image` (fall back to a generic placeholder SVG if `avatarUrl` is null), provides a hidden `<input type="file" accept="image/jpeg,image/png,image/webp">` triggered by clicking the avatar, on file selection sends a `multipart/form-data` POST to `/api/profile/avatar`, shows an inline loading spinner during upload, updates the displayed image on success, and shows an inline error message on failure
- [ ] Create `src/components/profile/NicknameForm.tsx` as a client component that renders a labeled text input pre-filled with the current nickname, a "저장" submit button, sends a PATCH request to `/api/profile` with `{ nickname }` on submit, disables the input and button while the request is in flight, shows a success toast or inline message on success, and shows an inline error message if validation fails (e.g., nickname too long) or the request errors
- [ ] Create `src/components/profile/PasswordForm.tsx` as a client component that renders three labeled password inputs — "현재 비밀번호", "새 비밀번호", "새 비밀번호 확인" — validates client-side that the two new-password fields match and that the new password is at least 8 characters before submitting, sends a POST to `/api/profile/password` with `{ currentPassword, newPassword }`, clears all fields on success, and displays specific inline error messages for wrong current password (401 from API) or other failures
- [ ] Add a `src/app/profile/loading.tsx` file that exports a skeleton loading UI matching the profile page layout (a circular skeleton for the avatar, two rectangular skeletons for the forms) so the page shows a placeholder while the server-side profile fetch resolves
- [ ] Add a `src/app/profile/error.tsx` client component that catches fetch errors from the profile page server component and displays a user-friendly error message with a "다시 시도" retry button that calls `reset()` from the error boundary props
