# Add Password Reset Backend (Token Utilities, Email, API Routes)

This todo implements the server-side logic for "비밀번호 찾기" on top of the `PasswordResetToken` table created in `01-password-reset-db-schema.md`. It covers token generation/validation utilities, an email-sending abstraction, and two Next.js Route Handlers. Assumptions: Nodemailer (or Resend if already present) for email via environment variables; `bcryptjs` for password hashing; `NEXT_PUBLIC_APP_URL` env var for constructing the reset link.

- [ ] Add the following environment variable keys to `.env.example` (do NOT set values in `.env`): `EMAIL_HOST`, `EMAIL_PORT`, `EMAIL_USER`, `EMAIL_PASS`, `EMAIL_FROM`, `NEXT_PUBLIC_APP_URL`. Document each with a comment explaining its purpose.
- [ ] Create a mailer utility at `src/lib/mailer.ts` that exports `sendPasswordResetEmail(email: string, resetUrl: string): Promise<void>`. Internally use `nodemailer.createTransport({ host, port, auth })` configured from the environment variables above. The email subject should be "비밀번호 재설정 링크" and the text body should include the `resetUrl` clearly. Throw a typed `MailerError` (extend `Error`) on transport failure so callers can distinguish mail failures from other errors.
- [ ] Create a token utility at `src/lib/password-reset-token.ts` that exports two functions: (a) `createResetToken(email: string): Promise<string>` — generates a token via `crypto.randomBytes(32).toString('hex')`, sets `expiresAt` to 1 hour from now, upserts (deletes any existing record for that email, then inserts) a `PasswordResetToken` row via Prisma, and returns the token string; (b) `validateResetToken(token: string): Promise<string | null>` — queries the record by `token`, returns `null` if not found, if `expiresAt` is in the past, or if `usedAt` is not null; otherwise returns the associated `email`.
- [ ] Create `src/app/api/auth/forgot-password/route.ts` as a Next.js Route Handler. Accept `POST { email: string }`. Validate email with a Zod schema (`z.string().email()`); return HTTP 400 `{ error: "유효한 이메일 주소를 입력해 주세요." }` on validation failure. Call `createResetToken(email)`, construct `resetUrl = \`${process.env.NEXT_PUBLIC_APP_URL}/reset-password?token=<token>\``, call `sendPasswordResetEmail(email, resetUrl)`. Always return HTTP 200 `{ message: "이메일을 확인해 주세요. 재설정 링크를 발송했습니다." }` regardless of whether the email is registered (prevents user enumeration). Catch and log mailer errors server-side without exposing them to the client.
- [ ] Create `src/app/api/auth/reset-password/route.ts` as a Next.js Route Handler. Accept `POST { token: string, password: string }`. Validate with Zod: `token` non-empty string, `password` string with minimum length 8; return HTTP 400 `{ error: "..." }` on failure. Call `validateResetToken(token)`; if null return HTTP 400 `{ error: "링크가 만료되었거나 유효하지 않습니다." }`. Otherwise hash the password with `bcryptjs` (`saltRounds: 12`), `UPDATE` the user row in the DB (`prisma.user.update({ where: { email }, data: { password: hash } })`), set `usedAt = new Date()` on the `PasswordResetToken` record, and return HTTP 200 `{ message: "비밀번호가 성공적으로 변경되었습니다." }`.
- [ ] Write unit tests in `src/lib/__tests__/password-reset-token.test.ts` covering: (a) `createResetToken` returns a non-empty hex string and the stored record has `expiresAt` roughly 1 hour in the future; (b) `validateResetToken` returns the correct email for a valid, unexpired, unused token; (c) `validateResetToken` returns `null` for a token whose `expiresAt` is in the past; (d) `validateResetToken` returns `null` for a token whose `usedAt` is set; (e) `validateResetToken` returns `null` for a token string that does not exist in the DB.
- [ ] Write integration tests for the API routes in `src/app/api/auth/__tests__/forgot-password.test.ts` and `reset-password.test.ts` covering: (a) `forgot-password` returns 400 on malformed email; (b) returns 200 with the standard message even when email is not in the DB; (c) `reset-password` returns 400 on missing/short password; (d) returns 400 on expired or unknown token; (e) returns 200 and updates the user password on a valid token.
