# Add Password Reset Backend (Token, Email, API)

This todo covers the entire backend layer needed to support "비밀번호 찾기" (forgot password). The feature requires generating and storing a time-limited reset token when a user requests a password reset, and sending that token to the user's email address as a clickable link. Assuming a Prisma ORM setup for token persistence and a Resend (or SMTP-compatible) email utility abstracted behind an environment variable — adjust to match the actual project's mailer/ORM if different.

- [ ] Add a `PasswordResetToken` model to `prisma/schema.prisma` with fields: `id` (cuid), `token` (String, unique), `email` (String), `expiresAt` (DateTime), `usedAt` (DateTime, optional), `createdAt` (DateTime). Run `npx prisma migrate dev --name add-password-reset-token` after adding the model.
- [ ] Create a reusable email utility at `src/lib/mailer.ts` that exports a `sendPasswordResetEmail(email: string, resetUrl: string): Promise<void>` function. Use `nodemailer` (or `resend` if already in the project) configured via `EMAIL_HOST`, `EMAIL_PORT`, `EMAIL_USER`, `EMAIL_PASS`, and `EMAIL_FROM` environment variables. Add a placeholder for the reset link in the email body, e.g. `"Click the link to reset your password: ${resetUrl}"`.
- [ ] Create a token utility at `src/lib/password-reset-token.ts` that exports: (a) `createResetToken(email: string): Promise<string>` — generates a cryptographically random token via `crypto.randomBytes(32).toString('hex')`, sets expiry to 1 hour from now, upserts a `PasswordResetToken` record for that email (invalidating any prior token), and returns the token string; (b) `validateResetToken(token: string): Promise<string | null>` — looks up the token, checks it is not expired and not already used, and returns the associated email or null.
- [ ] Create `src/app/api/auth/forgot-password/route.ts` as a Next.js Route Handler. Accept `POST` with body `{ email: string }`. Validate that email is non-empty and matches a basic email regex. Call `createResetToken(email)`, then call `sendPasswordResetEmail(email, resetUrl)` where `resetUrl` is `${process.env.NEXT_PUBLIC_APP_URL}/reset-password?token=<token>`. Always return HTTP 200 with `{ message: "If an account with that email exists, a reset link has been sent." }` regardless of whether the email is registered (prevents email enumeration).
- [ ] Create `src/app/api/auth/reset-password/route.ts` as a Next.js Route Handler. Accept `POST` with body `{ token: string, password: string }`. Validate that both fields are present and that password is at least 8 characters. Call `validateResetToken(token)` to get the email; if null/expired return HTTP 400 `{ error: "Invalid or expired token" }`. Otherwise hash the new password (use `bcryptjs` with saltRounds=12), update the user's password in the DB, mark the `PasswordResetToken` record as used by setting `usedAt = new Date()`, and return HTTP 200 `{ message: "Password updated successfully." }`.
- [ ] Add the required environment variables to `.env.example`: `EMAIL_HOST`, `EMAIL_PORT`, `EMAIL_USER`, `EMAIL_PASS`, `EMAIL_FROM`, `NEXT_PUBLIC_APP_URL`. Do NOT populate them in `.env` — only the example file.
- [ ] Write unit tests in `src/lib/__tests__/password-reset-token.test.ts` covering: (a) `createResetToken` returns a non-empty string and stores a record with a future expiry; (b) `validateResetToken` returns the email for a valid unexpired token; (c) `validateResetToken` returns null for an expired token; (d) `validateResetToken` returns null for an already-used token.
