# Add Password Reset UI (Login Link, Forgot Password Page, Reset Password Page)

This todo builds all UI changes for the "비밀번호 찾기" feature, depending on the API routes delivered in `02-password-reset-backend.md`. The flow is: login page shows "비밀번호 찾기" link → `/forgot-password` page with an email form → success message after submission; separately, the reset link in the email lands the user on `/reset-password?token=...` where they set a new password. Assuming Next.js App Router, Tailwind CSS for styling, and React Testing Library for component tests.

- [ ] In the existing login page component (likely `src/app/(auth)/login/page.tsx` or `src/components/auth/LoginForm.tsx`), add a Next.js `<Link href="/forgot-password">` element below the password input field. Style it as a small muted link: `text-sm text-muted-foreground underline hover:text-foreground`. The visible text should be "비밀번호를 잊으셨나요?", right-aligned or directly below the password field to match existing form layout.
- [ ] Create the page `src/app/forgot-password/page.tsx` as a server component with metadata `export const metadata = { title: "비밀번호 찾기" }`. It renders `<ForgotPasswordForm />` centered in the page inside a card or container consistent with the auth page layout (mirror the login page's surrounding layout/wrapper).
- [ ] Create the client component `src/components/auth/ForgotPasswordForm.tsx`. Props interface: none (self-contained). Internal state: `email: string`, `loading: boolean`, `success: boolean`, `error: string | null`. Render: (a) a labeled `<input type="email">` with label "이메일 주소" and placeholder "you@example.com"; (b) a submit button "재설정 링크 보내기" that is disabled and shows a spinner while `loading` is true; (c) on success, hide the form and show the message "이메일을 확인해 주세요. 재설정 링크를 발송했습니다."; (d) on error, show an inline error message below the input in red; (e) a "← 로그인으로 돌아가기" `<Link href="/login">` below the form.
- [ ] Create the page `src/app/reset-password/page.tsx` as a server component with metadata `export const metadata = { title: "비밀번호 재설정" }`. It wraps `<ResetPasswordForm />` in the same auth layout used by the forgot-password page. Since `useSearchParams` is needed in the form, wrap the page in a `<Suspense>` boundary to satisfy Next.js App Router requirements.
- [ ] Create the client component `src/components/auth/ResetPasswordForm.tsx`. Props interface: none. It reads the `token` query param via `useSearchParams()`. States: `password: string`, `confirmPassword: string`, `loading: boolean`, `success: boolean`, `error: string | null`. Behavior: (a) if `token` is absent or empty, immediately render "유효하지 않은 링크입니다. 이메일의 링크를 다시 확인해 주세요." without showing the form; (b) validate on submit that passwords match and are at least 8 characters — show "비밀번호가 일치하지 않습니다." or "비밀번호는 8자 이상이어야 합니다." as inline errors; (c) POST to `/api/auth/reset-password` with `{ token, password }`; (d) on success show "비밀번호가 변경되었습니다." and a `<Link href="/login">` "로그인하기" button; (e) on API 400 error show "링크가 만료되었거나 유효하지 않습니다. 다시 시도해 주세요."
- [ ] Write component tests in `src/components/auth/__tests__/ForgotPasswordForm.test.tsx` using React Testing Library covering: (a) renders the email input and "재설정 링크 보내기" button; (b) disables the submit button and shows a loading indicator while the fetch is in-flight (mock `fetch`); (c) hides the form and shows the success message on a mocked 200 response; (d) shows an error message below the input on a mocked non-200 response.
- [ ] Write component tests in `src/components/auth/__tests__/ResetPasswordForm.test.tsx` covering: (a) shows the "유효하지 않은 링크" message when no `token` param is present in the URL; (b) shows a validation error when the two password fields do not match; (c) shows a validation error when the password is fewer than 8 characters; (d) shows the success message and "로그인하기" link on a mocked 200 response; (e) shows the expired-token error message on a mocked 400 response.
