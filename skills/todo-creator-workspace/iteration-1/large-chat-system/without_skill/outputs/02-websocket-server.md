# Set up WebSocket server for real-time messaging

WebSocket is chosen over SSE because the chat feature requires bidirectional communication — clients must both send and receive messages in real time, and SSE is server-to-client only. This ticket sets up the WebSocket server infrastructure that the API layer and frontend will connect to. It integrates with the existing Next.js app using a custom server (`server.ts`) powered by the `ws` package.

- [ ] Install dependencies: `npm install ws` and dev dependencies `npm install -D @types/ws`
- [ ] Create `src/lib/websocket/types.ts` that exports a discriminated union type `WsMessage` covering all message shapes the server will send or receive. Initial variants: `{ type: 'chat:send', conversationId: string, content: string }`, `{ type: 'chat:message', message: MessageWithSender }` (outbound broadcast), `{ type: 'chat:read', conversationId: string, messageId: string }`, `{ type: 'error', code: string, detail: string }`. Export a `MessageWithSender` type that includes the `Message` fields plus a nested `sender: { id: string; name: string; image: string | null }`.
- [ ] Create `src/lib/websocket/server.ts` that exports a `createWsServer(httpServer: http.Server)` function. Inside, instantiate `new WebSocketServer({ server: httpServer })`. Maintain an in-memory `Map<string, Set<WebSocket>>` keyed by `conversationId` to track active connections per conversation room.
- [ ] In `src/lib/websocket/server.ts`, implement connection handling on `wss.on('connection', (ws, req) => { ... })`. Extract the authenticated user from the session cookie on the incoming `req` (reuse the project's existing auth session logic — read `src/lib/auth.ts` or equivalent to understand the session format). If unauthenticated, send `{ type: 'error', code: 'UNAUTHORIZED' }` and close the connection.
- [ ] Implement the `chat:send` handler: parse incoming JSON, validate the user is a member of the target `conversationId` (query `ConversationMember` via Prisma), persist the new `Message` to the database, then broadcast a `chat:message` event to all connected sockets in that conversation room. Include a `MessageWithSender` payload in the broadcast.
- [ ] Implement the `chat:read` handler: upsert a `MessageReadReceipt` row for `(messageId, userId)` and update `ConversationMember.lastReadAt` for `(conversationId, userId)`. Then broadcast a `{ type: 'chat:read', conversationId, messageId, userId }` event to all room members so clients can update their read-indicator UI.
- [ ] Handle connection cleanup: on `ws.on('close')`, remove the socket from all rooms it was subscribed to.
- [ ] Create `server.ts` in the project root (Next.js custom server). Import `http`, `next`, and `createWsServer`. Create the HTTP server with `http.createServer(handle)` where `handle` is the Next.js request handler. Call `createWsServer(httpServer)`. Listen on `process.env.PORT ?? 3000`. Update `package.json` `"dev"` script to `"ts-node server.ts"` and `"start"` to `"NODE_ENV=production ts-node server.ts"`. Install `ts-node` if not already present.
- [ ] Export a singleton Prisma client from `src/lib/prisma.ts` (if one does not already exist) to be used inside `src/lib/websocket/server.ts`.
