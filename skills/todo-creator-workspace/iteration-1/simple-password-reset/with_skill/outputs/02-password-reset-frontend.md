# Add Password Reset UI (Login Link, Forgot Password Page, Reset Page)

This todo covers all UI changes needed to expose "비밀번호 찾기" (forgot password) to users. It depends on the API routes created in `01-password-reset-backend.md`. The flow is: (1) user sees "비밀번호 찾기" link on the login page → (2) clicks it, lands on `/forgot-password` where they enter their email → (3) receives a success message; and separately (4) clicking the email link lands them on `/reset-password?token=...` where they enter a new password.

- [ ] In the existing login page component (likely `src/app/(auth)/login/page.tsx` or `src/components/auth/LoginForm.tsx`), add a "비밀번호 찾기" anchor/link below the password input field. Use Next.js `<Link href="/forgot-password">` styled as a small, muted text link (e.g., `text-sm text-muted-foreground underline`). Position it right-aligned or below the password field, matching the existing form's visual style.
- [ ] Create the page `src/app/forgot-password/page.tsx`. This is a server component page that simply renders the `<ForgotPasswordForm />` client component. Give it appropriate page metadata: `export const metadata = { title: "비밀번호 찾기" }`.
- [ ] Create the client component `src/components/auth/ForgotPasswordForm.tsx`. It should contain: (a) a controlled `<input type="email">` for the user's email address with a label "이메일 주소"; (b) a submit `<button>` labeled "재설정 링크 보내기"; (c) loading state (disable button and show spinner during the API call); (d) on success, hide the form and show a success message: "이메일을 확인해 주세요. 재설정 링크를 발송했습니다."; (e) on error, show an inline error message below the input. The form calls `POST /api/auth/forgot-password` with `{ email }` using `fetch`. Use `useState` for email, loading, success, and error states.
- [ ] Create the page `src/app/reset-password/page.tsx`. This is a server component that renders `<ResetPasswordForm />`. Add metadata: `export const metadata = { title: "비밀번호 재설정" }`.
- [ ] Create the client component `src/components/auth/ResetPasswordForm.tsx`. It should: (a) read the `token` query parameter from the URL using `useSearchParams()`; (b) if no token is present, display "유효하지 않은 링크입니다" error message; (c) show a form with a labeled `<input type="password">` for new password and a confirmation password input; (d) validate on submit that both passwords match and are at least 8 characters, showing inline error if not; (e) on submit, call `POST /api/auth/reset-password` with `{ token, password }`; (f) on success, show "비밀번호가 변경되었습니다." and a "로그인하기" link pointing to `/login`; (g) on API error (e.g., expired token), show "링크가 만료되었거나 유효하지 않습니다. 다시 시도해 주세요."
- [ ] Add a "로그인으로 돌아가기" (`← 로그인`) back-link to `src/components/auth/ForgotPasswordForm.tsx` below the form, using `<Link href="/login">`, so users can easily return without using the browser back button.
- [ ] Write integration/component tests in `src/components/auth/__tests__/ForgotPasswordForm.test.tsx` using React Testing Library covering: (a) renders the email input and submit button; (b) shows a loading state while the API call is in-flight; (c) shows the success message on a mocked 200 response; (d) shows an error message on a mocked non-200 response. Similarly add `ResetPasswordForm.test.tsx` covering: (a) shows an error when no token is in URL params; (b) shows validation error when passwords do not match; (c) shows success state on mocked 200 response; (d) shows expired-token error on mocked 400 response.
